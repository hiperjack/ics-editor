# ICS カレンダーエディタ

ローカルで動作する.icsファイルの編集ツールです。フライト等で開始/終了のタイムゾーンが異なるイベントに対応しています。

## 使い方

### オンラインで使う

GitHub Pagesで公開中: **https://hiperjack.github.io/ics-editor/**

ブラウザで開くだけでそのまま動作します。データはブラウザ内(localStorage)にのみ保存され、サーバーには一切送信されません。

### ローカルで使う

1. リポジトリから `ics-editor.html` をダウンロード
2. ダブルクリックして既定のブラウザで開く
3. 既存の.icsファイルを「ファイルを選択」から読み込むか、「空のカレンダーから始める」で新規作成
4. イベントカードをクリックで編集、「+ イベント追加」で新規追加
5. 「.icsダウンロード」で編集後のカレンダーをエクスポート

## 主な機能

- **ICSファイルの読み込み・編集・エクスポート**:すべてブラウザ内で完結、サーバー送信なし
- **タイムゾーン管理**:アフリカ・南北アメリカ・アジア・大西洋・オセアニア・欧州・太平洋の主要69タイムゾーンに対応(日本語名・UTC/JSTオフセット併記、UTCオフセット昇順)
- **表示TZの絞り込み**:ヘッダの「表示TZ設定」から、選択肢に出すTZを事前にチェックボックスで絞り込み可能(主要のみプリセットあり、使用中TZは自動で残す)
- **クロスタイムゾーン対応**:1つのイベントで開始(出発地)と終了(到着地)を別々のタイムゾーンに設定可能
- **所要時間の自動計算**:タイムゾーンを考慮した正確な所要時間表示
- **3つの表示モード**:日付別グルーピング、時刻軸付きカレンダー、Excel風テーブル直接編集
- **カレンダーの表示TZ切替**:表示中タイムゾーンに合わせて全イベントを変換配置(既定 JST)
- **Google/Outlook連携ボタン**:`.ics`ダウンロード＋各サービスのインポート画面を新規タブで自動オープン(ファイル選択のみで取り込み完了)
- **モバイル対応**:タブレット/スマートフォン(720px以下)ではカレンダーが単日縦スクロールに自動切替、前後日ナビ付き。タッチ向けボタンサイズ調整、ヘッダ開閉ボタンで縦領域を確保(テーブル編集はPC推奨のため非表示)
- **VALARM対応**:イベントごとにリマインダー(分前)を設定
- **自動保存**:編集内容はブラウザのlocalStorageに自動保存、再起動しても復元
- **ダーク/ライトモード自動切替**:OSのテーマに追従

## 動作環境

- Chrome、Edge、Safari、Firefox の現行バージョン
- インターネット接続不要(完全ローカル動作)
- ファイル単体で動作、依存なし

## 使い方のコツ

### フライトのイベント登録

国際線フライトを登録する場合は、モーダル内の「出発地と到着地でタイムゾーンが異なる」チェックボックスをONにします。

例)JL068 NRT 17:45発 → SEA 10:35着の場合:
- 開始日時: 2026-05-09 17:45 / 出発地TZ: Asia/Tokyo
- 終了日時: 2026-05-09 10:35 / 到着地TZ: America/Los_Angeles

カレンダーアプリにインポートすると、現地でカレンダーを開いた際に着陸時刻が「現地の時刻」で正しく表示されます。

### ICSのインポート先

ダウンロードした.icsファイルは以下にインポートできます:

- **iPhone/iPad**:ファイルをタップ → 「カレンダーに追加」
- **Mac**:ファイルをダブルクリック → カレンダーアプリが起動
- **Googleカレンダー**:ヘッダの「Googleで開く」をクリック → 設定の「インポート/エクスポート」画面でダウンロード済みファイルを選択
- **Outlook(業務 outlook.office.com)**:ヘッダの「Outlookで開く」をクリック → 「ファイルからアップロード」でダウンロード済みファイルを選択
- **Outlook(個人 outlook.live.com)**:設定 → カレンダー → 共有カレンダー → 「カレンダーをアップロード」(個人アカウント利用時はヘッダのリンクが業務アカウント向けのため、ここから手動で取込)

## 制限事項

- 繰り返しイベント(RRULE)は読み込めますが編集UIには未対応(出力時は失われます)
- 招待者(ATTENDEE)、添付ファイル(ATTACH)は未対応
- リマインダーは1イベントにつき1つまで(分単位指定)
- 終日イベント(VALUE=DATE)は時刻指定として扱われます
- 夏時間切替日にまたがるフライトの所要時間表示は1時間ずれる可能性あり(ファイル自体は正しいTZID参照を保持)
